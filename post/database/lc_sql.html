SQL

一、查询

查询的基本模板

在 WHERE 子句中作条件筛选

数值型范围：

  • [a,b] between a and b
  • 指定 set
    • in (1,2,3)
    • not in (1,2,3)

字符串型范围：

  • 指定相等 like "string"
  • 含通配符 like "%string"

通配符：% 匹配任意长度串，_ 匹配任意单个字符

去重与分组

DISTINCT 加在想要去重的 SELECT 选中的列名前面，表明该行希望去重

分组也带去重功能，但是可以按对该列每个唯一值单独进行一些其他统计

对结果行进行排序

order by 列名 ASC/DESC

选取有限结果行

limit k offset j 从第 j 行开始取 k 行

使用 JOIN 连接表

INNER JOIN：根据条件表达式，对两表做笛卡尔积连接 （可以简写为 JOIN）

LEFT JOIN：以左表为准 join（保留左表的所有行）

RIGHT JOIN：以右表为准 join （保留右表的所有行）

FULL JOIN：两个表的行都保留

关于 NULL

在一些NULL 表示它本来含义的场景，需要注意是否设置默认值还是保持NULL。 (f比如, 当你计算一些行的平均值的时候，如果是0会参与计算导致平均值差错，是NULL则不会参与计算).

判断字段是否为null，字段名 is null

统计函数

  • count()，注意 null 不参与计数
  • avg
  • max
  • min
  • sum
  • round，保留小数点后 k 位，round(num, k)

WHERE 与 GROUP BY HAVING

我们知道数据库是先对数据做WHERE，然后对结果做分组，如果我们要对分组完的数据再筛选出几条如何办？ （想一下按年份统计电影票房，要筛选出>100万的年份？）

一个不常用的语法 HAVING 语法将用来解决这个问题，他可以对分组之后的数据再做SELECT筛选.

WHERE 是用在对前面语句已经形成的表进行条件筛选的

但如果前面使用了 GROUP BY 进行分组统计，形成一个表之后还希望筛选，就需要使用 HAVING

HAVING 后写的条件，是对于分组后的每个组单独统计的

数据库在通过连接两张或多张表来返回记录时，都会生成一张中间的临时表，然后再将这张临时表返回给用户。 在使用left jion时，on和where条件的区别如下：

1、on条件是在生成临时表时使用的条件，它不管on中的条件是否为真，都会返回左边表中的记录。

2、where条件是在临时表生成好后，再对临时表进行过滤的条件。这时已经没有left join的含义（必须返回左边表的记录）了，条件不为真的就全部过滤掉。
