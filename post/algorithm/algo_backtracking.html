适用问题

回溯算法能解决如下问题：

  • 组合问题：N个数里面按一定规则找出k个数的集合
  • 排列问题：
  • 切割问题：
  • 子集问题：
  • 棋盘问题：N皇后，解数独等等
类型 描述 注意 特殊位置
组合 N 个数里面找出满足条件的 k 个数的组合 要返回的是集合，需要注意不能有重复（在形成集合的时候，一般通过控制单调来避免重复） 叶子节点（第 k 层）
排列 N 个数按一定规则全排列，有几种排列方式 返回的是序列而不是集合，不需要注意重复问题 叶子节点（第 N 层）
切割 切割一个字符串，符合条件的切割方案有几种 切割线位于串尾
子集 给出 N 个数的集合，求出所有符合条件的子集 每个节点
棋盘 求满足约束的所有棋盘布局 二维矩阵模拟+搜索

添加与撤回

回溯类问题，通常是需要记录遍历路径那类问题，在触达终点的时候，需要将当前记录的路径加入答案集合

这个”当前记录的路径“一般有几种形式：List 或者 String

我们知道，回溯是需要在搜索之后撤回一步修改的，这个撤回修改可以有两种实现形式

  • 只用一个序列，每次完成搜索后，删除序列末尾的元素（触底的时候只记录答案，不需要 return，否则会漏删最后的元素）
  • 每个函数调用都会开一个 frame，在每个栈帧中创建一个序列的副本，这样一来，函数调用退出时，就会回收这个变量，无需执行删除元素的步骤，缺点是浪费空间（触底的时候需要记录答案并 return）
